Ensuring visual consistency between platforms in the Product End-to-End suite

Getting Started

Overview

The purpose of this style guide is to define a visual standard for creation of PETE applications. The web based application uses the Bootstrap 3.0 framework for basic templating and structure. The following pages will convey examples of the graphic standards and provide codes snippets to help replicate the design specs.

  • Icon Set (Ionicons)
  • Bootstrap 3

Transitions and Interactions

Animations and Transitions

Animations give users visual feedback that helps them understand how they their actions have been processed. Simple hover animations should be used to show that an element can be activated or clicked. When elements move, they should not instantly appear in another location. Instead, their path should be animated in a fraction of a second. For larger movements (over 100px), the animation should be 0.8s and for subtle movements (less than 100px), transition duration should be 0.3s

Collapse and Expand Content

In an application, there may be areas where content needs to be hidden in order to remove clutter from the interface. The user may hide or reveal content by engaging the expand/hide icon. The icon is simply flipped 180° once it is toggled.

Toggle Snippet

Tabbed Content

When content needs to be seperated by tabs, use the default bootstrap component and add transitions to the active element which will shift to the right when it is clicked. A hover state (background-color) is also added in the style sheet.

Switches and Toggles

When there is a toggleable action that has two states (similar to a lightswitch), this iOS-inspired element can be used to indicate the two differnt states.

Draggable Elements

Draggable elements are widely used in the Global Assortment web application and may be found in other applications as well. There are three rules for making a draggable element. First, make sure that when an element is dragged, a copy is made rather than dragging the original element. Second, this element should be appended to the body so that it doesn't conflict with any restricts such as z-index or overflow. Third, if there is an intended "landing zone" for a draggable element, that area should be highlighted with an outline of 5px. A transition and inset box-shadow is added increase visibility

Toggle Snippet

Resizeable Elements

Too ensure that user's can maximize legibility and flexibility, users may be able to resize elements on the page. This functionality is very useful in the Pin View (Global Assortment). In Angular Apps, please use Angular-Resizable & AngularJS-Slider

Toggle Snippet

Dynamic Layouts

In applications with an abundunce of content, sections may be hidden to reduce clutter. Users can toggle hidden sections to reveal content. When a section is revealed, the rest of the page content is pushed to provide space for the new content.
